Enable all image fields in the current admin dashboard to accept images either from the existing Media Upload library or by pasting a direct URL (the current behavior).
Implement a clean, reusable Media Picker that fits perfectly into the existing UI/UX, without breaking anything or changing the database schema.

🔒 Core Rules


✅ Keep the existing stack (PHP + MySQL + Bootstrap).


✅ No database schema changes — images remain stored as URLs.


✅ The new Media Picker must look and behave consistently with the current dashboard design.


✅ It must integrate seamlessly — no redesigns or logic rewrites.


✅ All includes use absolute paths with __DIR__ to prevent require_once errors.


❌ Do not add external libraries.


❌ Do not break or rename any existing files or tables.



🧭 Where to Enable the Feature
Add a “Choose from Media” button next to every existing image URL input, at least in:


Site Settings (e.g., Logo URL).


Home / About section editors (any image fields).


Topics (hero image).


Content Items / Explainers (main article image).


The text field remains editable for manual URLs — the picker simply gives another way to fill it.

🎨 Expected User Experience


Clicking “Choose from Media” opens a small, clean interface (modal or off-canvas) that lists all uploaded images from the Media Upload section (already stored on the server).


Selecting an image automatically:


Fills the same input field with the image’s URL.


Displays a small preview beside or below the field.


Closes the modal.




Manual URL entry continues to work exactly as before.


The feature should match the dashboard’s style and color palette.



⚙️ Behavior Requirements


Display only images (filter by extension or MIME type).


Use simple pagination or lazy loading if there are many images.


No new tables — rely on the existing uploads storage mechanism.


If needed, reuse the current Media Upload directory or table to retrieve images.


Maintain all admin access and permissions logic exactly as it is.



🧱 Implementation Notes (conceptual)


The Media Picker should be implemented as a reusable component (e.g., a partial file or modal) that can be included on any admin page.


The component can dynamically populate the image list from the existing upload data.


The modal must be localized if bilingual text is active (support EN/AR labels if applicable).


All code additions must use the same naming and design conventions as the current admin system.



✅ Acceptance Criteria


A “Choose from Media” button appears next to every relevant image field.


Selecting an image fills the URL field with the proper path and updates a preview.


Manual URL entry still works.


No PHP or JS errors, and no broken includes.


No database schema changes.


The look and behavior are visually consistent with the existing dashboard.


The feature performs smoothly even with a large media library.
