
Build a fully functional, responsive Custom CMS (Content Management System) from scratch using PHP, MySQL, and Bootstrap 5, not WordPress.
The CMS will power a bilingual (English–Arabic) course website inspired by the IMP website design and structure.

🌐 FRONTEND (Public Website)
1. General Requirements

Use Bootstrap 5 for all responsive layouts and styling.

Include both English and Arabic versions (with language toggle in the header and footer).

Arabic version must support RTL layout (using a separate rtl.css or automatic direction detection).

Use placeholder images (e.g., https://via.placeholder.com/800x400) and placeholder text (Lorem Ipsum in both languages).

The website must be fully responsive (desktop, tablet, mobile).

All text content should come from the database, not hardcoded.

🧱 CMS STRUCTURE (Backend)
1. Admin Authentication

Login page (admin/login.php)

Email + Password fields

Remember me optional

Admin credentials stored in DB table admin_users

Passwords stored using password_hash()

Initial admin auto-generated at first setup (see setup logic below)

2. Admin Dashboard (admin/dashboard.php)

Accessible only after successful login (session-based authentication).

Sidebar navigation for all site sections:

Hero Section

About Course

Key Benefits

Course Details

About Us

Testimonials

FAQ

Contact Info

Site Settings (Logo, Site Name, Colors)

Media Upload

Manage Admins

Each section has a simple editable form (title, text, images, icons, etc.) that updates database records.

⚙️ FEATURES INSIDE ADMIN PANEL

Content Management

Edit all homepage sections directly.

Upload images (with preview and delete option).

All text fields and content stored in DB per language.

Option to add new courses or sections dynamically (scalable).

Media Upload System

Page admin/media_upload.php

“Upload Image” button → stores file in /uploads/ and saves path in DB.

Display gallery view of uploaded images with delete option.

Site Settings

Manage Logo (upload), Site Name, and color theme.

Saved in DB (e.g., table site_settings).

Manage Admins

Add, edit, or delete admin accounts.

Role = “Admin” (for now, single role system).

Password reset and change options.

Forgot Password

On the login page, include a “Forgot Password?” link.

Sends reset link via email (using SMTP if configured).

If SMTP not configured, show a temporary reset token in the console (for manual recovery).

Security

Sessions for login.

Auto logout after inactivity.

CSRF token on forms.

Escaped SQL queries (prepared statements).

Passwords hashed (no plain text).

🧰 INSTALLATION & DATABASE
1. Setup Script

Include a setup file (install.php or setup.php) that runs once:

Creates the MySQL tables automatically:

admin_users

hero_section, about_section, benefits, course_details, testimonials, faq, site_settings, uploads

Inserts one default admin user:

email: admin@example.com
password: password_hash('123456', PASSWORD_DEFAULT)


Displays these credentials in the console or first-time message (not stored in code).

Deletes itself or disables further access after installation.

2. Database Connection

includes/db.php handles all connections.

Database credentials configurable in one place.

3. Recovery

If admin credentials are lost, developer can manually reset via DB (replace password hash).
